## Per-item rundown

### P0-1 ‚Äî auth/session hang on reload  ‚úÖ
**Was:** hard-refresh on `/dashboard` etc. would stick on "Loading‚Ä¶" past the 5s console-warned navigator-lock acquisition, sometimes hard-cap-redirecting to `/login`. New tabs would bounce.
**Diagnosed:** React 19 StrictMode double-mounted useEffect raced `getSession()` against `onAuthStateChange`, both contending the same internal `_initialize()` lock. The 5s "soft" timeout left UI stuck because the recovery path waited on the same hung promise.
**Did:** `useAuth` is now subscribe-only ‚Äî it awaits `onAuthStateChange` INITIAL_SESSION (which the SDK fires with the persisted session as soon as `_initialize()` settles). 25s hard-cap timer kept as the truly-hung safety net. `supabase.ts` pins auth options explicitly so SDK defaults can't drift.
**Verified:** typecheck + build green. Behavioural smoke deferred to deploy ‚Äî see follow-ups.
**Commit:** `3ede608`
